# Task 2.3 - Create GTM audit rule catalog

## Status

Upcoming

## Goal

Create the initial audit rule catalog for GTM container review.

## How it fits

The audit rule catalog is the core knowledge source for identifying findings and proposed changes in the GTM Container Audit & Patch Package.

## In scope

- Define MLP audit rules for inventory, references, duplicates, unused entities, naming, ecommerce, consent, and risk.
- Include category, severity guidance, evidence requirements, and automated-change policy for each rule.
- Support rule IDs that audit findings can cite.

## Out of scope

- Implementing audit scripts.
- Running audits.
- Creating fixtures.
- Live GTM access.

## Definition of done for MLP

- [ ] Audit rule catalog exists.
- [ ] Each rule includes category, severity guidance, evidence requirements, and whether automated change is allowed.
- [ ] The Custom GPT can cite a rule ID in an audit finding.
