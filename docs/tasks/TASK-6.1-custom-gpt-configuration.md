# Task 6.1 - Create Custom GPT configuration

## Status

In Progress

## Goal

Create the Custom GPT configuration for early agency use.

## How it fits

The Custom GPT is the user interface for the MLP, while the repository and Skill remain the system of record.

## In scope

- Define GPT name and description.
- Configure intended internal access.
- Record configuration fields in repo files.
- Keep configuration aligned with the GTM Container Audit & Patch Package.

## Out of scope

- API actions.
- Live GTM access.
- Non-repo hidden source of truth.
- Publishing GTM changes.

## Definition of done for MLP

- [ ] GPT exists.
- [ ] GPT has a clear name and description.
- [ ] GPT is accessible to intended internal users.
- [ ] GPT has version history through the GPT editor flow.
