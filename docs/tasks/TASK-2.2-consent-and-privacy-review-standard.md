# Task 2.2 - Create consent and privacy review standard

## Status

Upcoming

## Goal

Define how the MLP should flag consent-sensitive tags and changes.

## How it fits

The MLP is file-in/file-out and cannot certify that consent works on site. It should flag risk, not claim legal compliance.

## In scope

- Define consent-sensitive vendors and tag types.
- Require manual review for Custom HTML.
- Require review for tags that appear to transmit sensitive user data.
- Define wording that avoids legal compliance claims.

## Out of scope

- Legal advice.
- Privacy certification.
- Live consent testing.
- API connectors.

## Definition of done for MLP

- [ ] Consent standard exists.
- [ ] Risky vendors are listed.
- [ ] Custom HTML is always flagged.
- [ ] Output wording avoids legal compliance claims.
