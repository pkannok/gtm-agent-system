# Task 5.1 - Create synthetic GTM container fixtures

## Status

Complete

## Goal

Create controlled synthetic GTM container export fixtures with known issues.

## How it fits

Fixtures let the MLP workflow be tested against repeatable inputs without using real client containers.

## In scope

- Create small GTM export fixtures.
- Cover representative issue types.
- Document expected findings.
- Avoid real client data.

## Out of scope

- Large fixture library.
- Live GTM exports.
- Automated fixture generator.
- Production test harness beyond active task scope.

## Definition of done for MLP

- [x] At least six synthetic fixtures exist.

- [x] Each fixture has known expected issues.
- [x] No fixture contains real client data.
