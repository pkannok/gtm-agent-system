# Glossary

## GTM Container Audit & Patch Package

The canonical MLP deliverable. It includes optimized GTM JSON plus audit, change log, validation, QA, and metadata artifacts for analyst review.

## Custom GPT

The user-facing interface for the workflow. It should reference repo-owned instructions and deliverable definitions rather than becoming the only source of truth.

## Skill

A future reusable package that will reference the same deliverable contract and supporting documentation.

## MLP

Minimum lovable product. In this project, the MLP is a useful file-in/file-out package for GTM container audit and review.

## Source Of Truth

Version-controlled repository files that define project decisions, task scope, deliverable contracts, and reusable instructions.

## `optimized_container.json`

One artifact in the GTM Container Audit & Patch Package. It is not the entire deliverable and must not be treated as safe to publish without human review.
